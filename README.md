## 0. Что такое .NET и CLI-команды `dotnet`

**.NET** — это бесплатная кроссплатформенная платформа от Microsoft для создания приложений. Она включает в себя среду выполнения (Runtime), которая запускает ваш код, и библиотеки.

Управление проектами происходит через консольный интерфейс — **C# CLI** (Command Line Interface). Все команды начинаются со слова `dotnet`.

### Основные терминальные команды

Выполняйте эти команды в терминале, находясь в папке вашего проекта:

*   `dotnet new console` — Создать новый пустой проект консольного приложения в текущей папке.
*   `dotnet build` — Скомпилировать (собрать) проект. Проверяет код на ошибки и создает исполняемый файл в папке `bin/`.
*   `dotnet run` — Скомпилировать и сразу запустить проект. Самая частая команда при разработке.
*   `dotnet watch` — Запустить проект в режиме «живой перезагрузки». Программа сама перезапустится, как только вы сохраните изменения в коде.
*   `dotnet clean` — Очистить все временные файлы сборки (удаляет папки `bin/` и `obj/`). Полезно, если проект ведет себя странно и «глючит».

### Структура проекта (что создают эти команды)

Когда вы пишете `dotnet new console`, C# создает несколько важных файлов:
*   `Program.cs` — Главный файл, где пишется ваш код.
*   `ИмяПроекта.csproj` — Главный файл настроек проекта (XML-формат). Там указана версия .NET и подключенные сторонние библиотеки. Никогда не удаляйте его.

## 1. Структура проекта и консоль

Минимальный шаблон программы и базовый ввод-вывод данных.

```csharp
using System; // Подключение пространства имен

namespace ConsoleApp {
    class Program {
        static void Main(string[] args) {
            // Вывод текста с переносом строки и без
            Console.WriteLine("Hello World!"); 
            Console.Write("Введите что-нибудь: ");

            // Чтение строки из консоли
            string input = Console.ReadLine(); 
            Console.WriteLine("Вы ввели: " + input);

            // Ожидание нажатия одной клавиши (чтобы консоль не закрывалась)
            Console.ReadKey(); 
        }
    }
}
```

## 2. Типы данных и переменные

В C# жесткая типизация. Каждому типу выделяется строгое количество байт в памяти.

### Целочисленные (Integer)

| Тип     | Размер  | Диапазон / Описание                       |
| :------ | :------ | :---------------------------------------- |
| `byte`  | 1 байт  | от 0 до 255                               |
| `short` | 2 байта | от -32 768 до 32 767                      |
| `int`   | 4 байта | от -2.14 млрд до 2.14 млрд (основной тип) |
| `uint`  | 4 байта | от 0 до 4.29 млрд (только положительные)  |
| `long`  | 8 байт  | огромные целые числа                      |
| `ulong` | 8 байт  | огромные только положительные целые числа |

### Числа с плавающей точкой (Дробные)
*   `float`: 4 байта. Требует суффикс `f` при объявлении (`1.6f`).
*   `double`: 8 байт. Самый используемый тип для дробей (`32.12`).

### Символы, Текст и Логика
*   `char`: Один символ в одинарных кавычках (`'A'`).
*   `string`: Строка (список символов) в двойных кавычках (`"name"`).
*   `bool`: Логический тип, принимает только `true` или `false`.

### Преобразование типов и dynamic
```csharp
// Конвертация строки в число
int n1 = Convert.ToInt32(Console.ReadLine());
double n2 = double.Parse(Console.ReadLine());

// dynamic — тип, меняющий свой тип на лету во время выполнения
dynamic dyn = 0;      // Сейчас это int
dyn = "name";         // Теперь это string
Console.WriteLine(dyn.ToUpper()); // Выведет: NAME
```

## 3. Арифметика и класс Math

### Операторы и инкремент
```csharp
double result = (v + 10d - 10) * 2 / 10 % 1.23; // % — остаток от деления

result += 5; // Сокращенная запись (result = result + 5)

// Инкремент и декремент
result++; // Постфиксный (увеличение после использования)
++result; // Префиксный (увеличение до использования)
```

### Класс Math (Встроенные математические функции)
```csharp
double pi = Math.PI;          // Число Пи (~3.14159)
int abs = Math.Abs(-10);      // Модуль числа (10)

Math.Ceiling(4.11);           // Округление вверх (= 5)
Math.Floor(4.99);             // Округление вниз (= 4)
Math.Round(4.5);              // Округление по правилам математики (= 4 или 5)

// Также доступны: Math.Sin(), Math.Cos(), Math.Sqrt() (корень), Math.Pow() (степень), Math.Min(), Math.Max()
```

### Форматирование и интерполяция строк (Плейсхолдеры)

// Подстановка переменных по индексам {0}, {1}

Console.WriteLine($"Еще один {"Cпособ".ToLower()}");

## 4. Условные операторы (if / switch)

### Конструкция if-else и Логические И/ИЛИ
```csharp
int a = int.Parse(Console.ReadLine());

if (a == 1) {
    Console.WriteLine("a равно 1");
} else if (a > 1) {
    Console.WriteLine("a больше 1");
} else {
    Console.WriteLine("a меньше 1");
}

bool b = false;
// && — Логическое И (AND)
// || — Логическое ИЛИ (OR)
// !  — Отрицание НЕ (NOT)
if ((b || !b) && (a > 1)) {
    Console.WriteLine("Условие выполнилось");
}
```

### Конструкция switch-case
Удобна, когда нужно перебирать конкретные значения одной переменной.
```csharp
switch (a) {
    case 1:
        Console.WriteLine("Ветка 1");
        break; // break обязателен!
    case 0:
        Console.WriteLine("Ветка 0");
        break;
    default:
        Console.WriteLine("Если ни один case не подошел");
        break;
}
```

## 5. Массивы (Одномерные и Многомерные)

Массивы имеют **фиксированную** длину, которую нельзя изменить после создания.

### Одномерные массивы
```csharp
// Способ 1: Выделение памяти, заполнение по индексам
byte[] aboba = new byte[4]; // Массив на 4 элемента (индексы 0, 1, 2, 3)
aboba[0] = 1;
aboba[1] = 2;

// Способ 2: Инициализация сразу с элементами
string[] names = new string[] { "aboba", "goida" };

// Перебор массива через обычный цикл for
for (int i = 0; i < names.Length; i++) {
    Console.WriteLine(names[i]);
}
```

### Генерация случайных чисел в массив
```csharp
Random random = new Random();
short[] numbers = new short[10];

for (int i = 0; i < numbers.Length; i++) {
    numbers[i] = Convert.ToInt16(random.Next(5, 15)); // Числа от 5 до 14 включительно
}
```

### Двумерные (Многомерные) массивы `[,]`
```csharp
// Матрица из 2 строк и 3 колонок
char[,] symbols = new char[2, 3]; 

// Инициализация со значениями напрямую:
int[,] nums = {
    {1, 2, 3}, // Строка 0
    {4, 5, 6}, // Строка 1
    {7, 8, 9}  // Строка 2
};

// Вывод двумерного массива на экран через вложенные циклы
for (int i = 0; i < 2; i++) {
    for (int j = 0; j < 3; j++) {
        Console.Write(symbols[i, j] + " ");
    }
    Console.WriteLine(); // Перенос на новую строку после завершения ряда
}
```

## 6. Цикл foreach

Используется для быстрого и безопасного чтения всех элементов коллекции. Изменять элементы коллекции внутри `foreach` **нельзя**.

```csharp
short[] fornums = new short[] { 1, 2, 3, 4 };

foreach (short num in fornums) {
    Console.WriteLine(num); // num — копия текущего элемента
}

// Перебор многомерного массива через foreach разворачивает его в одну сплошную линию
foreach (short num in nums) {
    Console.WriteLine(num); // Выведет последовательно: 1, 2, 3, 4, 5...
}
```

## 7. Динамические списки (List)

Списки находятся в пространстве `using System.Collections.Generic;`. В отличие от массивов, они могут динамически расширяться (менять размер).

```csharp
// Создание списка целых чисел
List<int> list = new List<int>() { 1, 2, 3 };

list.Add(4);       // Добавит 4 в конец списка -> [1, 2, 3, 4]
list.Remove(1);    // Удалит ПЕРВОЕ встреченное значение "1" -> [2, 3, 4]
list.RemoveAt(0);  // Удалит элемент по ИНДЕКСУ 0 (это двойка) -> [3, 4]

// Быстрый вывод списка на экран
foreach (int item in list) {
    Console.Write(item + " ");
}

// Полезные встроенные методы:
// list.Sort();    // Сортировка по возрастанию
// list.Reverse(); // Перевернуть список задом наперед
// list.Count;     // Получить текущее количество элементов (аналог .Length у массивов)
```

## 8. Функции (Методы) и Перегрузка

Функции в C# пишутся внутри классов. Для использования в `Main` без создания объектов класса они должны иметь модификатор `static`.

```csharp
class Program {
    static void Main(string[] args) {
        Console.WriteLine(Aboba());        // Выведет: string
        Console.WriteLine(Aboba("test"));  // Выведет: testtest

        byte[] array = { 1, 2, 3 };
        Console.WriteLine(Summa(array));   // Выведет: 6
    }

    // public — метод доступен везде
    // static — принадлежит самому классу, а не его экземплярам
    // string — тип возвращаемого значения (void, если ничего не возвращает)
    public static string Aboba() {
        return "string";
    }

    // ПЕРЕГРУЗКА МЕТОДА: имя то же, но другие входные аргументы
    public static string Aboba(string arg) {
        return arg + arg;
    }

    // Передача массива в качестве аргумента
    public static int Summa(byte[] array) {
        int res = 0;
        foreach (int elem in array) {
            res += elem;
        }
        return res;
    }
}
```

## 9. Строки и Работа с Файлами (I/O)

### Полезные методы строк (String)
Строка — это неизменяемый (immutable) массив символов `char`.

```csharp
string s = "aboba";
Console.WriteLine(s[2]); // Доступ по индексу -> 'o'

string people = "a, b, o";
// Разделение строки в массив по разделителю
string[] parts = people.Split(", "); 
// Сборка массива обратно в строку с новым разделителем -> "a/b/o"
Console.WriteLine(String.Join("/", parts)); 

Console.WriteLine("   3   0   ".Trim()); // Удаляет пробелы в начале и конце -> "3   0"
Console.WriteLine("1234".Substring(2, 2)); // Вырезает подстроку (с какого индекса, сколько символов) -> "34"

// s.ToUpper() — в верхний регистр, s.ToLower() — в нижний
```

### Запись и Чтение файлов через FileStream
Использование конструкции `using` автоматически закрывает файл и освобождает память (стрим) после выхода из блока `{}`.

```csharp
using System.IO;
using System.Text;

// ЗАПИСЬ: Создает или перезаписывает файл text.txt
using (FileStream stream = new FileStream("text.txt", FileMode.OpenOrCreate)) {
    // Перевод строки в массив байт в стандартной кодировке
    byte[] array = Encoding.Default.GetBytes("Данные для записи");
    stream.Write(array, 0, array.Length);
}

// ЧТЕНИЕ: Безопасный способ вычитать файл целиком
using (FileStream stream = File.OpenRead("text.txt")) {
    byte[] array = new byte[stream.Length];
    
    // .ReadExactly гарантирует, что буфер заполнится полностью, исключая баги незавершенного чтения
    stream.ReadExactly(array, 0, array.Length);

    string text = Encoding.Default.GetString(array);
    Console.WriteLine(text);
}
```

## 10. Обработка Исключений (Exceptions)

Используется для защиты программы от аварийного завершения (падения) при ошибках (например, неверный ввод пользователя).

```csharp
int a = -1;
try {
    // Код, который может вызвать ошибку
    a = Convert.ToInt16(Console.ReadLine());
} 
catch (FormatException ex) {
    // Сработает ТОЛЬКО если пользователь ввел буквы вместо числа
    Console.WriteLine("Вы ввели не число!");
}
catch (Exception e) {
    // Сработает на любую другую ошибку. e.Message — текст системной ошибки
    Console.WriteLine(e.Message);
} 
finally {
    // Этот блок выполнится ВСЕГДА (была ошибка или нет). Удобно для очистки ресурсов
    Console.WriteLine("Блок try-catch завершен");
}
```

## 11. Основы ООП: Классы, Свойства и Наследование

ООП в C# во многом повторяет Java, но имеет уникальный синтаксис для работы с полями (инкапсуляцией) и вызова родительских конструкторов.

### Модификаторы доступа
*   `private` — доступ только внутри текущего класса (используется для скрытия полей).
*   `protected` — доступ внутри текущего класса **и всех его наследников**.
*   `public` — доступ открыт для всех в проекте.
*   По умолчанию (если ничего не написать) члены класса получают доступ `private`.

### Обычные методы vs Свойства (Properties)
В C# принято скрывать поля через `private`, но вместо написания Java-подобных методов `getData()` и `setData()` используются **Свойства** с блоками `get` и `set`.

```csharp
class NewClass {
    // Внутреннее приватное поле
    private byte[] data;

    // Вариант 1: Классические геттер и сеттер (как в Java)
    public byte[] GetData() { return data; }
    public void SetData(byte[] data) { this.data = data; }

    // Вариант 2: Свойство C# (Удобный геттер и сеттер)
    public byte[] Data {
        get { return this.data; }
        set { this.data = value; } // value — это то, что передают в свойство
    }

    // Вариант 3: Автосвойство (C# создает скрытое поле за вас автоматически!)
    // public byte[] AutoData { get; set; }
    
    // Вариант 4: Ограничение доступа (читать можно всем, менять только внутри класса)
    // public byte[] SafeData { get; private set; }
}

// Использование в коде:
NewClass myClass = new NewClass();
myClass.Data = new byte[] { 1, 2, 3 }; // Автоматически вызовется блок set (value)
Console.WriteLine(myClass.Data[0]);     // Автоматически вызовется блок get
```

### Ключевое слово static
*   `static` переменная или метод принадлежат **всему классу**, а не конкретным объектам. Нам не нужно писать `new Robot()`, чтобы обратиться к ним.
*   Статические методы могут работать **только со статическими** переменными класса.

```csharp
class Robot {
    public static string emotion = "hate..."; // Одна переменная на весь класс
    public string name = "amoruch";            // У каждого робота свое имя
}

// Использование:
string mood = Robot.emotion; // Вызов напрямую через имя КЛАССА
Robot myRobot = new Robot();
string hisName = myRobot.name; // Вызов через Экземпляр (объект)
```

### Конструкторы и Наследование
C# поддерживает **только одинарное наследование** (класс может наследоваться только от одного родителя). Класс-наследник перенимает все поля и методы родителя (кроме приватных).

Вместо Java-вского слова `super`, в C# используется ключевое слово `base`.

```csharp
// Базовый (Родительский) класс
class Robot {
    public string name;

    public Robot() { } // Конструктор по умолчанию
    public Robot(string name) {
        this.name = name;
    }
}

// Класс-наследник (NGRobot наследуется от Robot через двоеточие ":")
class NGRobot : Robot {
    public string parent = "ozymandias";

    // Вызов конструктора родителя происходит через ": base(параметры)" перед телом метода
    public NGRobot(string name) : base(name) {
        Console.WriteLine(\$"Робот '{name}' успешно создан.");
        
        // Также внутри методов можно вызывать родительские методы: base.SomeMethod();
    }
}
```

## 12. Приведение типов (is / as) и Цепочки конструкторов

Когда у вас есть иерархия классов (например, родитель `Element` и наследник `Ilement`), объекты могут динамически приводиться к разным типам в процессе работы.

### Оператор `as` (Безопасное приведение)
Пытается привести объект к указанному типу. Если приведение **возможно**, возвращает объект этого типа. Если **невозможно**, программа не падает с ошибкой, а просто возвращает `null`.

```csharp
foreach (Ilement elem in lest) {
    // Так как Ilement унаследован от Element, его можно безопасно трактовать как Element
    Element e = elem as Element; 
    
    if (e != null) {
        Console.WriteLine(e.ToString());
    }
}
```

### Оператор `is` (Проверка типа)
Проверяет, является ли объект экземпляром определенного типа (или его наследником). Возвращает логическое значение `true` или `false`.

```csharp
if (lest[0] is Element) {
    // Переменная lest[0] действительно является (или наследуется от) Element
    Console.WriteLine("Объект относится к типу Element");
}
```

> **Фишка современного C# (Pattern Matching):** 
> Вы можете проверить тип и сразу записать его в новую переменную в одну строку:
> ```csharp
> if (lest[0] is Element validElement) {
>     Console.WriteLine(validElement.Name); // validElement доступен внутри этого блока if
> }
> ```

### Переопределение методов (`override`)
Чтобы изменить стандартное поведение встроенных функций C#, используется ключевое слово `override`. Самый частый пример — метод `ToString()`, который вызывается автоматически, когда вы передаете объект в `Console.WriteLine()`.

```csharp
class Element {
    public string Name { get; private set; }
    public string info = "aboba";

    // Переопределяем стандартный метод вывода объекта в виде строки
    public override string ToString() {
        return Name + ":" + info;
    }
}
```

### Цепочки конструкторов через `this()`
Если в вашем классе несколько конструкторов, вы можете вызвать один конструктор из другого с помощью ключевого слова `: this()`, чтобы не дублировать код инициализации.

```csharp
class Ilement : Element {
    public string More { get; set; }

    // Конструктор 1 (по умолчанию)
    public Ilement() : base("Ilement") {
        More = "idk";
        Console.WriteLine("Вызвался первый конструктор");
    }

    // Конструктор 2 (с параметром)
    // Сначала отработает Конструктор 1 через : this(), а затем выполнится код внутри этого блока
    public Ilement(string more) : this() {
        More = more; // Перезаписываем значение
        Console.WriteLine("Вызвался второй конструктор");
    }
}
```

## 13. Полиморфизм: Виртуальные методы (virtual / override)

Полиморфизм позволяет наследникам изменять поведение методов родительского класса. В C# для этого используется связка ключевых слов `virtual` и `override`.

### Как это работает:
1.  **`virtual`** — ставится у метода в **родительском** классе. Это явное разрешение для наследников: «Этот метод можно переписать».
2.  **`override`** — ставится у метода в **классе-наследнике**. Означает: «Я переписываю логику этого родительского метода».

```csharp
class MyClass {
    // Разрешаем переопределение метода
    public virtual void Method() {
        Console.WriteLine("Делаем базовые вещи");
    }
}

class ChildClass : MyClass {
    // Переопределяем метод родителя
    public override void Method() {
        Console.WriteLine("Делаем совершенно другие вещи");
        
        // Фишка: Если нужно выполнить базовый код И добавить свой, 
        // можно вызвать родительский метод через base:
        // base.Method();
    }
}
```

### В чем отличие от Java?
В Java все методы классов по умолчанию виртуальные (их можно переопределить в любой момент). В C# методы по умолчанию **закрыты** для переопределения. Если вы забудете написать `virtual` в базовом классе, то при попытке написать `override` в наследнике компилятор выдаст ошибку.

## 14. Абстрактные классы и методы (abstract)

**Абстрактный класс** — это класс-шаблон, на основе которого **нельзя** создать объект напрямую (`new Animal()` вызовет ошибку). Он нужен исключительно для того, чтобы быть базовым классом для других наследников.

### Основные правила:
1.  Если класс помечен как `abstract`, внутри него можно создавать **абстрактные методы** (`public abstract void MakeSound();`).
2.  Абстрактный метод **не имеет тела** (кода внутри `{}`). Это просто декларация: «Все наследники обязаны уметь это делать».
3.  Обычные классы-наследники **обязаны** реализовать все абстрактные методы родителя, используя ключевое слово `override`.
4.  Внутри абстрактного класса можно писать и **обычные методы** с готовой реализацией (например, `Sleep()`). Наследники получат их автоматически.

```csharp
// Базовый шаблон для всех животных
abstract class Animal {
    // 1. Обычный метод (реализация одинакова для всех)
    public void Sleep() {
        Console.WriteLine("Животное спит... zzz");
    }

    // 2. Абстрактный метод (каждое животное реализует его по-своему)
    public abstract void MakeSound();
}

// Конкретный класс-наследник
class Dog : Animal {
    // Обязательная реализация звука для собаки
    public override void MakeSound() {
        Console.WriteLine("Гав-гав!");
    }
}
```

### Главная польза: Полиморфные коллекции
Абстрактный класс позволяет объединять разные объекты в один список и управлять ими централизованно.

```csharp
// Создаем список типа Animal, но кладем туда разных наследников (Dog, Cat, Horse)
List<Animal> animals = new List<Animal>() {
    new Dog(),
    // new Cat(), 
    // new Horse()
};

foreach (Animal animal in animals) {
    animal.MakeSound(); // Каждый объект издаст свой уникальный звук!
}
```

## 15. Интерфейсы (interface)

**Интерфейс** — это строгий контракт для класса. Он описывает только *что* класс должен уметь делать, но не говорит *как*. Интерфейсы не содержат логики (кода) и полей для хранения данных, но могут содержать свойства и сигнатуры методов.

### Основные правила:
1.  **Именование**: Имя любого интерфейса в C# всегда начинается с заглавной буквы `I` (`IRun`, `IJump`, `ICloneable`).
2.  **Множественность**: Класс может наследоваться только от *одного* класса, но может реализовывать *любое количество* интерфейсов через запятую.
3.  **Модификаторы**: Внутри интерфейса методы и свойства не требуют модификатора доступа (по умолчанию они `public`). При реализации в классе они обязательно должны быть помечены как `public`.

```csharp
// Описание контракта "Умеет бегать"
interface IRun {
    float speed { get; set; } // Описание свойства
    void Run();               // Описание метода (без тела {})
}

// Описание контракта "Умеет прыгать"
interface IJump {
    float y { get; set; }
    void Jump();
}

// Класс реализует сразу ДВА интерфейса (множественная реализация)
class Athlet : IRun, IJump {
    // Обязаны реализовать все свойства из интерфейсов
    public float speed { get; set; }
    public float y { get; set; }

    // Обязаны реализовать все методы из интерфейсов с модификатором public
    public void Run() {
        Console.WriteLine("Атлет бежит со скоростью " + speed);
    }

    public void Jump() {
        Console.WriteLine("Атлет прыгает на высоту " + y);
    }
}
```

### В чем разница между Абстрактным классом и Интерфейсом?
*   **Абстрактный класс** используется, когда между классами есть связь «является частью чего-то» (*Собака является Животным*). Он может иметь готовый код и внутреннее состояние (поля).
*   **Интерфейс** используется, когда нужно добавить общую способность для абсолютно разных классов «умеет делать это» (*Робот, Человек и Машина — все умеют реализовывать `IRun`*).

## 16. Перечисления (enum)

**Перечисление (`enum`)** — это набор именованных констант, который заменяет «магические числа» понятными словами. По умолчанию каждому элементу присваивается число от `0` и далее по порядку, а базовым типом является `int`.

### Объявление и явные значения
```csharp
// 1. Обычное перечисление (индексы: 0, 1, 2)
enum Weapon {
    aboba, 
    sword, 
    spear 
}

// 2. С явным указанием числовых значений (например, коды ответов сервера)
enum HttpStatusCode {
    Ok = 200,
    BadRequest = 400,
    Unauthorized = 401,
    NotFound = 404,
    InternalServerError = 500
}

// 3. Смена базового типа данных (целочисленного) для экономии памяти
enum GameDifficulty : byte {
    Easy,   // 0
    Medium, // 1
    Hard    // 2
}
```

### Работа с enum (Приведение типов, Парсинг и Перебор)
```csharp
// Использование и приведение к числу (casting)
Weapon weapon = Weapon.aboba;
Console.WriteLine((int)weapon); // Выведет числовой индекс: 0
Console.WriteLine(weapon.ToString()); // Выведет строковое имя: "aboba"

// Безопасный парсинг строки в enum через 'out'-переменную
string input = "aboba";
// Если строка совпадет с одним из элементов, TryParse вернет true и запишет результат в переменную result
if (Enum.TryParse<Weapon>(input, out Weapon result)) { 
    Console.WriteLine(\$"Успешно распарсено: {result}");
}

// Получение списка всех элементов перечисления
string[] names = Enum.GetNames<Weapon>();      // Массив строк ["aboba", "sword", "spear"]
Weapon[] values = Enum.GetValues<Weapon>();    // Массив объектов [Weapon.aboba, Weapon.sword, ...]

foreach (Weapon w in values) {
    Console.WriteLine(w);
}
```
